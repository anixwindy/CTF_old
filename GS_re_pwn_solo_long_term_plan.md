# RE -> Pwn Solo Long-Term Plan

適用對象：

- 單人
- 業餘
- 長期慢慢走
- 主線是 `RE -> Pwn`
- `Crypto` 只補常見 CTF 生存能力

這份筆記把三件事放在一起：

1. 半年到一年以上的實際學習順序
2. 每個階段學到什麼程度才算能往下
3. 單人長期版該走什麼，不該太早分心去碰什麼

---

## 1. 先講時間觀念

不要把月份當死線，要把它當節奏參考。

大致上：

- 如果你每週大約 `5` 到 `8` 小時：這份路線通常會走到 `10` 到 `14` 個月
- 如果你每週大約 `10` 到 `15` 小時：通常可壓到 `6` 到 `10` 個月

真正應該決定你能不能往下的，不是日曆，而是：

- 你能不能看懂
- 你能不能驗證
- 你能不能講清楚

---

## 2. 你的主引擎：一路都在練的能力

你不是在收集題型，而是在建立四個母能力：

1. 觀察  
   找輸入點、比較點、常數、字串、資料流、控制流、保護機制。

2. 翻譯  
   把 `C`、組語、反編譯、記憶體內容互相對照。

3. 驗證  
   用最小步驟驗證假設，例如先確認一個 offset、先確認一個比較、先確認 leak 指到了哪裡。

4. 表達  
   能說清楚「它在做什麼」「為什麼這是洞」「為什麼這個 leak 有價值」。

`RE` 強化前 1 到 3 點，`Pwn` 把前 1 到 4 點全部逼到更扎實。

---

## 3. 單人長期版的總原則

1. 永遠先學 `C`、記憶體、組語，再學 exploit。
2. 永遠先做靜態觀察，再決定要不要動態分析。
3. 永遠先講清楚漏洞模型，再寫 payload。
4. 永遠先走 `x86-64 Linux` 主線，不要太早分散到多架構。
5. `Crypto` 只維持基本辨識與解題能力，不走重理論。
6. 同一時間只保留一條主線，不要同時開 `RE`、`Heap`、`Windows malware`、`Kernel` 四條支線。

---

## 4. 半年到一年以上的實際順序

下面這個版本是最適合單人、業餘、長期走的主線。

### Phase 0：起手環境與學習習慣

建議時間：

- `1` 週左右

你要建立：

- `Linux` 練習環境
- `gcc` / `clang`
- `gdb`
- `Ghidra`
- `file` `strings` `xxd` `readelf` `objdump`
- 基本筆記習慣

這階段要做的不是解題，而是把最小工具鏈裝好，並養成固定流程：

1. 先看檔案資訊
2. 再看字串與常數
3. 再看反編譯或組語
4. 最後才進 `gdb`

可以往下的標準：

- 你知道每個工具大概是幹嘛的
- 你能自己編譯一個很小的 C 程式並用 `gdb` 跑起來

這階段不要分心：

- 不要急著碰題目平台大量刷題
- 不要急著學 `pwntools`

---

### Phase 1：`C` 語言行為與 Linux 基礎

建議時間：

- 第 `1` 到第 `2` 個月

這是之後全部分析的底層。

你要學的知識：

- 整數型別
- signed / unsigned
- cast、截斷、overflow
- pointer 與 array
- string 與 `\\0`
- `struct` / padding
- stack 上的局部變數
- 函式參數與回傳
- `malloc` / `free`
- `memcpy` `strcpy` `printf` 類函式的風險

你要學的技術：

- 自己寫很小的 C 範例驗證觀念
- 用編譯器產出組語
- 看小型 C 程式如何落到記憶體與 stack

你要會用的工具：

- `gcc`
- `objdump`
- `gdb`
- `xxd`

這階段的核心問題：

- 一段 C 編譯後大概會長怎樣
- 一個危險函式為什麼危險
- 一個 buffer 為什麼會被寫爆

可以往下的標準：

- 你能畫出小函式的 stack frame 粗略圖
- 你能解釋 `printf(user_input)` 為什麼危險
- 你能解釋為什麼陣列越界可能踩到別的資料
- 你看到簡單的 C 片段時，不會只停在語法層

如果還沒到這個程度，不要急著碰 `Pwn`。

---

### Phase 2：`x86-64`、呼叫慣例、`gdb` 基礎

建議時間：

- 第 `2` 到第 `3` 個月

你要學的知識：

- `mov` `lea` `cmp` `test`
- `add` `sub` `xor`
- `push` `pop` `call` `ret`
- `jmp` 與條件跳轉
- flag register 基本作用
- `System V AMD64 ABI`
- 參數寄存器
- 回傳值位置
- `rbp` / `rsp`
- prologue / epilogue

你要學的技術：

- 從組語反推大概的 C 結構
- 在 `gdb` 中追 register 變化
- 在關鍵 `cmp` 前後停下來看比較值

你要會用的工具：

- `gdb`
- `objdump -d`
- 編譯器輸出的組語

可以往下的標準：

- 給你一個很小的函式組語，你能指出參數、局部變數、回傳邏輯
- 你能說出某個 `cmp` 後面為什麼跳成功或失敗分支
- 你能用 `gdb` 找到某個輸入在 register 或 stack 上的大概位置

這階段不要分心：

- 不要急著碰多架構
- 不要急著背一大堆冷門指令

---

### Phase 3：`RE` 第一階段，靜態分析主導

建議時間：

- 第 `3` 到第 `5` 個月

這一段開始真正進入 `RE`。

你要學的知識：

- `ELF` 基本辨識
- `file`、`strings`、`readelf` 提供的資訊
- 匯入函式、符號、常數、字串、table 的觀察方法
- Ghidra/IDA 的基本視角

你要學的技術：

- 不執行 binary 也先做初步判斷
- 先找輸入點、比較點、輸出點
- 先看固定字串與 magic value
- 先從反編譯看輪廓，再回組語確認關鍵點

你常見會遇到的模式：

- 字串比對
- XOR / ADD / SUB
- 分段驗證
- switch / jump table
- checksum
- table lookup

可以往下的標準：

- 拿到一個小 crackme，你能先不跑它就說出大致驗證流程
- 你能指出成功條件大概在哪一段
- 你能分辨「這是直接字串比對」還是「先轉換再比對」

這階段不要分心：

- 不要一開始就衝動態分析
- 不要看到題目就直接暴力

---

### Phase 4：`RE` 第二階段，動態驗證與控制流理解

建議時間：

- 第 `5` 到第 `7` 個月

這一段是把你從「看大概」拉到「能證明」。

你要學的知識：

- register 與 stack 觀察
- 條件比較前後的值追蹤
- 函式之間資料怎麼傳
- 反編譯與真實控制流落差

你要學的技術：

- 精準下中斷點
- 單步追關鍵函式
- 驗證某個猜測是否正確
- 追一個 byte 如何被轉換後拿去比較

你應該開始碰的 RE 題型：

- 多函式拆散驗證
- 簡單混淆
- table-based 驗證
- 小型 state machine
- 一點點 anti-debug 觀察

可以往下的標準：

- 你不只會說「感覺是 XOR」，而是能指出哪裡 XOR、key 是什麼、比對點在哪
- 你能在 `gdb` 中證明某段輸入如何變成最終比較值
- 你能解釋反編譯看起來怪的地方，實際組語在做什麼

這階段不要分心：

- 不要急著碰惡意程式分析
- 不要急著碰大型 C++ binary

---

### Phase 5：`ELF`、連結、保護機制，為 `Pwn` 鋪路

建議時間：

- 第 `6` 到第 `8` 個月

這一段本質上還是在做 `RE`，但目的是把你接到 `Pwn`。

你要學的知識：

- section 與 segment
- symbol
- relocation
- `PLT` / `GOT`
- 動態連結大概在做什麼
- libc 基本角色
- `NX` `Canary` `PIE` `ASLR` `RELRO`

你要學的技術：

- 先看保護機制再猜利用方向
- 看懂某個函式呼叫是經過 `PLT`
- 初步理解 leak 為什麼能推回 base

可以往下的標準：

- 給你一個 binary，你能先講它的保護組合
- 你能說出 `NX` 防的是什麼
- 你能說出 `PIE` 和 `ASLR` 為什麼讓位址不穩
- 你能大致講清楚 `PLT/GOT` 為什麼會在 `Pwn` 很重要

這階段不要分心：

- 不要直接跳進 `heap`
- 不要急著背 exploit 模板

---

### Phase 6：`Pwn` 第一階段，Stack 模型與 `ret2win`

建議時間：

- 第 `7` 到第 `9` 個月

你現在才正式進 `Pwn` 主線。

你要學的知識：

- stack overflow
- local buffer、saved `rbp`、return address 的相對位置
- offset 概念
- 為什麼 overflow 會改控制流

你要學的技術：

- 找 crash 點
- 找 offset
- 驗證 `RIP` 是否可控
- 做最簡單的 `ret2win`

你要會用的工具：

- `gdb`
- `checksec`
- cyclic pattern 類工具

可以往下的標準：

- 你能拿到一個簡單 stack overflow 題，自己找出偏移量
- 你能證明 return address 被覆蓋了
- 你能自己說明 exploit 的成功點，不只是 payload 能跑

這階段不要分心：

- 不要先學 shellcode 細節當主線
- 不要因為看到 `pwntools` 很方便就跳過模型理解

---

### Phase 7：`ROP`、leak、`ret2libc`

建議時間：

- 第 `9` 到第 `11` 個月

這是第一個真正的 `Pwn` 分水嶺。

你要學的知識：

- 為什麼 `NX` 讓你不能單純執行 stack 上資料
- gadget 是什麼
- 為什麼 `ROP` 能串出邏輯
- libc 位址洩漏
- base 計算
- `system("/bin/sh")` 的基本套路

你要學的技術：

- 找一個可用 leak
- 從 leak 算出 libc base
- 找 gadget
- 構造最小可行 `ROP chain`

你要會用的工具：

- `pwntools`
- `ROPgadget` 或同類工具
- `pwndbg` 類輔助插件

可以往下的標準：

- 你能自己說清楚為什麼這題需要 leak
- 你能自己算一次 libc base，而不是只抄腳本
- 你能指出每個 gadget 在 ROP 鏈中的用途
- 你能解釋為什麼 payload 長這樣

這階段不要分心：

- 不要太早去做極度 gadget-heavy 的怪題
- 不要因為 exploit 模板可跑就以為觀念穩了

---

### Phase 8：Format String 與 `Heap` 入門

建議時間：

- 第 `11` 到第 `14` 個月

這一段才開始碰較難的 `Pwn` 分支。

你要學的知識：

- 為什麼 `printf(user_input)` 是 format string
- 為什麼可以 leak stack/libc/pie
- `%n` 類型寫入的本質
- `malloc` / `free` 背後 allocator 的基本想法
- chunk header
- `tcache`
- double free
- use-after-free

你要學的技術：

- 分辨 format string 題主要是拿 leak 還是拿 write
- 初步觀察 chunk 流向
- 驗證某個 freed chunk 為什麼又被拿回來

可以往下的標準：

- 你能解釋 format string 題中每一步為什麼有價值
- 你能看懂一個小 heap 題中 chunk 大概怎麼流動
- 你理解 heap 初期重點是模型，不是背版本招式

這階段不要分心：

- 不要太早鑽很版本依賴的 advanced heap 技巧
- 不要把所有時間都丟進 heap

## 5. 一年後怎麼接著走

如果你照這條主線走到一年左右，下一步不要同時開太多分支。

最穩的順序是：

1. 先補強 `RE` 深度
2. 再補強 `Pwn` 難題
3. 最後才選擇要不要開平台或架構支線

### 一年後先補強的 `RE`

優先補這些：

- C++ 物件模型
- vtable
- RTTI
- STL 常見編譯後痕跡
- stripped binary 的閱讀
- 編譯器優化造成的反編譯扭曲

原因：

- 這些會大幅提升你看真實 binary 的能力
- 也會讓你之後分析較複雜題目時不會完全依賴反編譯器

### 一年後再補強的 `Pwn`

優先順序建議：

1. 更穩的 `ROP`
2. 更穩的 leak 與資訊推回
3. format string 進階
4. heap 深一點
5. 視題目需要再碰 `ret2dlresolve`、`seccomp`、`SROP`

原因：

- 這樣投報比最高
- 單人路線最怕同時碰太多零碎技巧但沒有一條真的打通

## 6. 單人業餘版，哪些支線不要太早開

下面這些不是永遠不學，而是不要太早碰。

### 先延後的 `RE` 支線

- `Windows PE` 深入
- 惡意程式分析
- packer / unpacking
- `ARM` / `MIPS` / `RISC-V` 多架構同步學
- kernel reversing

原因：

- 這些很吃背景知識
- 對你當前 `RE -> Pwn` 主線投報比不高

### 先延後的 `Pwn` 支線

- kernel pwn
- browser exploitation
- hypervisor / sandbox escape
- 高版本特化 heap 技巧大全
- 一開始就追很花的 CTF 特殊技巧

原因：

- 對單人業餘來說，容易投入很大但主線不紮實

### 先維持生存等級的 `Crypto`

只補這些：

- Base16 / Base32 / Base64
- XOR
- Caesar / ROT
- Vigenere
- substitution
- hash 辨識
- 模運算基本概念
- 非常入門的 RSA 題型辨識

不要太早走的方向：

- 抽象代數深水區
- 橢圓曲線深理論
- 大型密碼學證明導向內容

## 7. 每個階段常見卡點

### `C` 階段常見卡點

- 把 pointer 和 array 當一樣
- 不理解 `\\0`
- 不理解 signed / unsigned 比較差異
- 不理解 local variable 在 stack 的相對位置

### 組語階段常見卡點

- 只背指令，不理解資料流
- 不知道比較前的值從哪裡來
- 看到 `lea` 就以為只是取地址

### `RE` 階段常見卡點

- 過度相信反編譯
- 不做靜態觀察就急著跑 binary
- 一看到 encoding 或 bitwise 就直接暴力

### `Pwn` 階段常見卡點

- 只記 payload 模板
- 不先看保護機制
- 不理解 leak 的價值
- 太早去碰 heap

## 8. 最適合單人長期走的每週節奏

如果你不是全職，建議用固定小節奏，而不是爆衝再斷掉。

一個很穩的比例是：

- `2` 次：主線學習  
  例如本週只學 stack frame，或只學 `PLT/GOT`

- `1` 次：小練習  
  拿一個最小 binary 驗證這週觀念

- `1` 次：整理筆記  
  把「我怎麼知道的」寫下來，不只寫答案

這樣比一週亂碰五種主題有效得多。

## 9. 你什麼時候才算真的能往下

請用這個原則，不要只看「有沒有做出題」。

### 可以從 `C` 往組語走

當你能：

- 解釋 pointer / array / string / stack 的基本行為
- 用自己的話說明幾個危險函式的風險

### 可以從組語往 `RE` 走

當你能：

- 看懂小函式的流程
- 指出比較點與回傳邏輯
- 用 `gdb` 追一個變數

### 可以從 `RE` 往 `Pwn` 走

當你能：

- 講出 binary 的輸入點與關鍵比較點
- 畫出粗略 stack layout
- 解釋保護機制如何影響利用方向

### 可以從基礎 `Pwn` 往進階 `Pwn` 走

當你能：

- 自己找 offset
- 自己解釋 leak 為什麼成立
- 自己講清楚一條簡單 `ROP` 鏈

### 可以從 stack 類往 `Heap` 走

當你能：

- 理解記憶體 primitive 的價值
- 能把 leak、base、write 之間的關係講清楚
- 不會把每題都當成單純 overflow 題

- 最後學會利用

## 11. 最後給你的實際建議

如果你是單人、業餘、打算一輩子慢慢走，最穩的比重通常是：

- `RE`：`60%`
- `Pwn`：`30%`
- `Crypto`：`10%`

原因不是 `Pwn` 不重要，而是：

- `RE` 是你的主引擎
- `Pwn` 一直在吃 `RE` 能力
- `Crypto` 對你來說是生存技能，不是主專精

所以你的長期目標應該寫成一句話：

**我不是要背最多技巧，而是要把 binary 的行為、控制流、資料流、保護機制、漏洞效果講清楚。**

做到這一點，你的 `RE` 和 `Pwn` 會一起長。
